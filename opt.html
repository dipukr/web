  • Avoid allocating objects within loops.
  • Avoid creating an object only used for accessing a method.
  • Use buffer IO and tune the buffer size.
  • Minimize the contention for shared resources.
  • Keep synchronized methods out of loops.
  • Minimize the creation of short-lived objects.
  • Use the int type instead of the long type whenever possible.
  • Avoid remote method calls.
  • Use callbacks to avoid blocking remote method calls.
  • Avoid invoking the garbage collector unless necessary.
  • You should invoke GC only during idle time or some noncritical phase.
  • Group native operations to reduce the number of JNI calls when possible.
  • Reorder classpath so that the most frequently used libraries occur first.
  • Store string and char data as unicode in the database.
  • Use connection pools and cached-prepared statements for database access.
  • Avoid the costs of object creation and manipulation by using primitive variables.
  • Avoid unnecessary casts and instanceof references as casting is done at runtime.
  • Use synchronized methods only when necessary to limit the multitasking in JVM.
  • Use the static final when creating constants to reduce repeated vars initialization.
